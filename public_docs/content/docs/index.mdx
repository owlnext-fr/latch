`latch` is a small Rust app that serves single-file HTML prototypes behind a controlled
host, with versioning and an optional access code per project. It runs as a **single
[Loco](https://loco.rs) (axum) binary** with **SQLite bundled in** — no Redis, no workers,
no external services.

## The three surfaces

A single binary exposes three surfaces, each with its own audience:

<Cards>
  <Card title="Client serving — /c/<slug>" href="/docs/admin/access-codes">
    Serves the active version of a prototype. Free projects are served directly; protected
    ones show a styled unlock page (6-digit PIN) before the HTML. Everything is sent
    `Cache-Control: no-store`.
  </Card>
  <Card title="Admin — /admin" href="/docs/admin/projects">
    A React SPA behind a session cookie: projects, versions, manual deploy, active-version
    switching, and access-code configuration.
  </Card>
  <Card title="MCP — /mcp" href="/docs/publish-from-claude/connect-mcp">
    A Model Context Protocol endpoint that Claude calls. Three tools —
    `deploy_prototype`, `list_projects`, and `pull_prototype` — all guarded by a deploy token.
  </Card>
</Cards>

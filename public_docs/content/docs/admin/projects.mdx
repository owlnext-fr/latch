---

The admin lives at `/admin` behind a session cookie. Sign in with the `ADMIN_USER` /
`ADMIN_PASS` credentials, and you land on the project list.

![The latch admin project list](/img/admin-list.png)

## Create a project

Click **New project** to open a side-panel. You set:

- **Name** (required) — what the project is called in the admin.
- **Slug** — a readable base derived from the name plus a random 8-character suffix (for example
  `my-project-k7Qp2maZ`). It's shown **read-only** — the random suffix keeps the public URL
  practically non-guessable.
- **Access code** — on by default, with an auto-generated 6-digit PIN. See
  [Access codes](/docs/admin/access-codes).
- **Brand name** — optional, shown on the unlock page. See
  [Co-branding](/docs/admin/co-branding).

<Callout type="info">
The slug's base is read-only in this version. The public link is `/<base-url>/c/<slug>`.
</Callout>

## The project list

Each row in the project list includes a **Preview** action (icon button). It opens the project's
currently active version in a new browser tab through an admin-only, `no-store` route, so you can
check the live prototype without leaving the list. The action is unavailable when the project has
no active version yet.

The **access code** column shows a lock badge: a closed padlock when the project is protected by a
code, an open padlock when it is freely accessible.

## The detail view

Clicking a project opens its detail page, which is **read-only** — every edit happens in a
dedicated side-panel. From here you:

- copy the public URL (and reveal/copy the PIN if the project is protected),
- **Edit** the project (name, code, brand name),
- **Deploy** a new version,
- **Delete** the project (a danger side-panel with a typed confirmation),
- manage versions — see [Versions](/docs/admin/versions).

## Comments

The project edit panel exposes a **`comments_enabled`** toggle. Its default depends on
whether the project has an access code:

| Access code | Default for `comments_enabled` |
| --- | --- |
| Enabled (PIN required) | **Off** — comments disabled until explicitly turned on. |
| Disabled (open access) | **On** — comments open for writing, protected by an anti-spam filter. |

This default is security-aware: a publicly accessible prototype gets comments automatically;
a gated prototype keeps comments off until the admin intentionally enables them.

When enabled, reviewers see a floating action bar on the prototype that lets them anchor
comments to specific elements. See [Commenting a prototype](/docs/admin/comments) for details.

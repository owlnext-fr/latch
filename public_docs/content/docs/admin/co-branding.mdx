---
title: Co-branding
description: Put a brand name on the unlock page.
---

A protected project can carry an optional **brand name**, shown on the unlock page so the client
sees a prototype prepared for them rather than a bare PIN prompt.

## Set it

Open the project's **Edit** side-panel and fill in **Brand name**. It's plain text — no logo
upload. Leave it empty to show the default unlock copy.

When set, the unlock page reads along the lines of *"Prototype prepared for …"* with the brand
name filled in.

<Callout type="info">
The brand name is purely cosmetic and only appears on the unlock page of a **protected** project.
It does not change the slug, the URL, or any access rule.
</Callout>

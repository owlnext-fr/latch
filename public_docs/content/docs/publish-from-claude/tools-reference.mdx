title: Tools reference
description: The two MCP tools — deploy_prototype and list_projects.
---

`latch` exposes exactly two MCP tools. Both validate the `deploy_token` **first**, before any
database access. Neither ever returns a hash or a PIN.

## `deploy_prototype`

Deploys a single-file HTML prototype as a new version of an existing project.

```text
deploy_prototype(slug, html, deploy_token, activate?, release_notes?)
```

| Argument | Notes |
| --- | --- |
| `slug` | Must reference an **existing** project — there is no auto-creation. An unknown slug is an error. |
| `html` | The single-file HTML prototype. |
| `deploy_token` | Validated first, in constant time. |
| `activate` | Optional, defaults to **`true`** — the new version becomes active immediately. |
| `release_notes` | Optional. Lightweight Markdown (see below). Max **10 000 characters** — exceeding this returns a validation error. |

### `release_notes` format

Pass a short Markdown string to describe what changed in this version. Supported elements:
headings, bold, italic, unordered lists, ordered lists, blockquotes.

**Not rendered on the client:** links, images, code blocks, and raw HTML are stripped by the
restricted renderer — use plain descriptive text only.

```text
## What's new in v3

- Revised onboarding flow
- Fixed contrast on dark mode
- Updated hero copy
```

On success it returns:

```json
{
  "url": "https://latch.example.com/c/my-project-k7Qp2maZ",
  "version": 2,
  "code_protected": true
}
```

The `url` is built from `LATCH_PUBLIC_BASE_URL`. **No PIN, no hash** is ever included.

<Callout type="info">
When a visitor opens the URL for the first time after you deploy with release notes, they see an
overlay with the notes. Dismissing it stores the version number locally — the overlay won't
reappear for the same version.
</Callout>

## `list_projects`

Lists the projects so you can pick a slug to deploy to.

```text
list_projects(deploy_token)
```

It returns an **object envelope** (not a bare array):

```json
{
  "projects": [
    {
      "slug": "my-project-k7Qp2maZ",
      "name": "My project",
      "code_protected": true,
      "active_version": 2
    }
  ]
}
```

Each entry is a summary — `slug`, `name`, `code_protected`, `active_version`. **No PIN, no hash,
no database id.**

<Callout type="info">
Configuring access codes, switching the active version, and deleting are **admin-only** — they are
intentionally never exposed over MCP. See the [admin guide](/docs/admin/projects).
</Callout>
